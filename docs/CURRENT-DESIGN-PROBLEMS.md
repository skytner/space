# Проблемы текущего дизайна Space

Краткий разбор того, что сейчас не так с навигацией и структурой экранов. Нужно для выравнивания видения (UI/UX, архитектура, продукт).

---

## 1. Дублирование пути к карте

- **Сайдбар:** пункт «Map» ведёт на `/map`.
- **Корень `/` (General):** экран с карточками; единственная карточка — «Map» с кнопкой «Open map» → `/map`.

Итог: два способа попасть на одну и ту же карту. Выбор «General vs Map» в сайдбаре не несёт смысла: General лишь предлагает перейти на Map.

---

## 2. Неверная точка входа

- Приложение клиентское, главный инструмент — **карта** (2D, pan, инерция).
- Сейчас по умолчанию открывается **General** (карточки), карта — на втором клике.

Ожидание: открыл приложение → сразу карта. Реальность: открыл → экран с одной карточкой → клик → карта. Лишний шаг и размытый фокус продукта.

---

## 3. «General» без ясной роли

- В навигации есть «General» и «Map».
- General = фон (звёзды) + заголовок «Space» + одна карточка «Map».

Непонятно, зачем отдельный раздел «General»: дашборда нет, списка разделов по сути нет (есть только ссылка на карту, которая уже есть в сайдбаре). Раздел создаёт шум и ощущение «лендинга» вместо «инструмента».

---

## 4. Расхождение с заявленным видением

- В `docs/UI-UX-VISION.md`: корень `/` = карта, один экран — карта + сайдбар, без карточек на главной.
- В коде: `/` = HomeView с карточками, `/map` = карта.

Архитектура роутов и контента не совпадает с описанным UX-видением.

---

## 5. Итог в одном предложении

**Главный экран — не карта; путь к карте продублирован (сайдбар + карточка); раздел General не даёт пользы и размывает фокус.** Исправление: сделать карту корневым экраном (`/`), убрать или переосмыслить General и экран с карточками.
